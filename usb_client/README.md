# Moto IMPRES USB-клієнт як `.exe`

`client_usb.html` сам по собі потребує **Chrome/Edge** (Web Serial API) і не завжди
працює з `file://`. Цей міст усуває обидва обмеження: він відкриває COM-порт
**напряму** (через `pyserial`) і показує той самий інтерфейс у **будь-якому**
браузері через локальний веб-сервер `http://127.0.0.1:8765`.

Зібраний `moto_usb.exe` **не потребує** ні Python, ні Chrome у користувача.

## Запуск із коду (для розробки)

```bat
pip install pyserial
python moto_bridge.py            REM автовибір єдиного COM-порту
python moto_bridge.py --port COM5
```

Відкриється браузер. Оберіть COM-порт у списку зверху (якщо їх кілька) і
натисніть **«Підключити»**.

## Збірка одного `.exe`

На машині з Python 3.8+:

```bat
build.bat
```

Готовий файл — `dist\moto_usb.exe` (≈8–12 МБ, самодостатній: усередині вже
вкладено `client_usb.html`). Копіюйте його куди завгодно й запускайте подвійним
кліком — відкриється браузер із інтерфейсом.

Ручний варіант тієї ж збірки:

```bat
pip install pyserial pyinstaller
pyinstaller --onefile --name moto_usb --add-data "..\client_usb.html;." moto_bridge.py
```

## Як це працює

```
браузер (будь-який)  --HTTP-->  moto_usb.exe (Python-міст)  --COM/UART-->  ESP32
        \_ той самий client_usb.html, лише cmd() йде через /cmd, а не Web Serial
```

Міст строго послідовний: на кожну команду надсилає рядок `CMD\n` і чекає рівно
одну відповідь `#R#{json}` (відладкові рядки ESP ігноруються). Пароль по USB —
опційний (як і в браузерній версії).

## Примітки

- Порт зайнятий, якщо відкритий Arduino Serial Monitor — закрийте його.
- Драйвер USB-UART (CP210x / CH340 / FTDI) має бути встановлений — тоді
  з'явиться `COMx`.
- Антивірус іноді підозріло дивиться на свіжозібрані PyInstaller-`.exe` (хибне
  спрацювання). Збирайте самі з цього коду, якщо потрібна довіра.
- Це той самий протокол, що описаний у `serial_api.h`; міст нічого не змінює в
  логіці — лише переносить транспорт із Web Serial у нативний COM.

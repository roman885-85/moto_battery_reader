# Moto IMPRES USB-клієнт як `.exe`

Два варіанти, обидва відкривають COM-порт **напряму** (через `pyserial`), тож
**не потребують Chrome/Web Serial**:

| Файл | Що це | Збірка |
|---|---|---|
| **`moto_gui.py`** | **Нативний GUI** (Tkinter), власне вікно, **без браузера** ⭐ | `build.bat` → `dist\moto_usb.exe` |
| `moto_bridge.py` | Веб-міст: той самий `client_usb.html` у будь-якому браузері | `build_bridge.bat` → `dist\moto_usb_bridge.exe` |

Зібраний `.exe` **не потребує** ні Python, ні браузера у користувача.

## Нативний GUI (рекомендовано)

Запуск із коду:

```bat
pip install pyserial
python moto_gui.py
```

Збірка одного `.exe` (без консольного вікна) на машині з Python 3.8+:

```bat
build.bat
```

Готовий файл — `dist\moto_usb.exe` (≈9–13 МБ, самодостатній). Копіюйте куди
завгодно, запускайте подвійним кліком.

**Робота:** оберіть COM-порт у списку вгорі → **«Підключити»** (ESP32
перезавантажується ~2 с) → вкладки:
- **Огляд** — заряд, напруга, температура, модель, ємність/знос, цикли, справжність, цілісність;
- **Дані** — серійний (ROM DS2438), DS2438-телеметрія, hex-дампи;
- **Прошивка** — копії `.bin`, скидання, ремонт, запис моделі, **Новий акумулятор**
  (список вшитих моделей + мА·год), заряд/здоров'я, очистка, запис `.bin`,
  повне стирання, перезавантаження;
- **Редактор** — правка байтів DS2433/DS2438 з автовиправленням сум.

Пароль (поле вгорі) **опційний** — по USB запис працює й без нього.

## Веб-міст (альтернатива)

Якщо зручніше звичний веб-інтерфейс, але в будь-якому браузері:

```bat
python moto_bridge.py            REM автовибір єдиного COM-порту
python moto_bridge.py --port COM5
build_bridge.bat                 REM -> dist\moto_usb_bridge.exe
```

Відкриє `http://127.0.0.1:8765/` у браузері за замовчуванням; COM-порт обробляє
Python, тож Web Serial не потрібен.

## Примітки

- Порт зайнятий, якщо відкритий Arduino Serial Monitor — закрийте його.
- Драйвер USB-UART (CP210x / CH340 / FTDI) має бути встановлений — тоді
  з'явиться `COMx`.
- Свіжозібрані PyInstaller-`.exe` іноді викликають хибне спрацювання антивірусів;
  за потреби збирайте самі з цього коду.
- Протокол — той самий, що в `serial_api.h`; клієнт лише переносить транспорт
  із Web Serial у нативний COM.

# Motorola IMPRES Battery Reader (ESP32) — повна інструкція

Пристрій для **зчитування, аналізу, редагування та програмування** чипів
акумуляторів Motorola IMPRES через веб-інтерфейс Wi-Fi і OLED/LCD-дисплей.

- **DS2433** — EEPROM 512 байт (модель, крива розряду, історія ємності,
  лічильники, блок автентифікації).
- **DS2438** — монітор батареї (напруга, струм, температура, лічильники
  заряду/розряду ICA/CCA/DCA, калібрування).

Інтерфейс українською, заставка з емблемою Національної Гвардії України.

> ⚠️ **1-Wire family codes для ЦЬОГО заліза — `DS2433_ID = 0xA3`,
> `DS2438_ID = 0xA6`** (перевірено осцилографом, у `battery_reader.h`). Це
> НЕ стандартні 0x23/0x26 з даташита. Заміна ламає зчитування — не чіпати.

---

## 1. Що потрібно

**Плата:** будь-яка ESP32 (пакет `esp32` від Espressif, перевірено на 3.3.10).
Прошивка займає ~80 % флеша — підходить схема розділів за замовчуванням.

**Бібліотеки** (Arduino Library Manager):
- `U8g2` (oliver) — дисплей
- `OneWire` — шина 1-Wire
- `ArduinoJson`

## 2. Підключення

| Сигнал | GPIO | Константа | Примітка |
|---|---|---|---|
| 1-Wire дані (DS2433/DS2438) | **13** | `DS_PIN` | до контакту даних АКБ |
| Керування підтяжкою 1-Wire | **12** | `PULLUP_PIN` | сильний pull-up на час запису (tPROG) |
| LED зелений | **14** | `LED_GREEN_PIN` | через резистор ~330 Ом на GND |
| LED червоний | **27** | `LED_RED_PIN` | через резистор ~330 Ом на GND |
| Кнопка «Вперед» | **25** | `MENU_BTN_PIN` | між GPIO і GND (активний LOW) |
| Кнопка «Назад» | **26** | `MENU_BTN2_PIN` | між GPIO і GND (активний LOW) |

Усі піни змінюються у `settings.h`.

**1-Wire:** між лінією даних (GPIO13) і 3.3V потрібна підтяжка ~2.2…4.7 кОм.
`PULLUP_PIN` короткочасно дає сильну підтяжку під час програмування EEPROM.

## 3. Вибір дисплея

У `settings.h`, **КРОК 1** — розкоментуйте **рівно один** рядок:

```c
#define DISPLAY_SSD1306_I2C        // 0.96" 128x64 I2C (GME12864) — за замовч.
// #define DISPLAY_SH1106_I2C      // 1.3"  128x64 I2C
// #define DISPLAY_SH1107_128_I2C  // GME128128-02, 128x128 I2C (варіант SH1107)
// #define DISPLAY_SSD1327_128_I2C // GME128128-02, 128x128 I2C (варіант SSD1327)
// #define DISPLAY_ST7567_SPI      // Open-Smart 1.8" 128x64 SPI
// #define DISPLAY_PCD8544_SPI     // Nokia 5110 (PCD8544) 84x48 SPI
```

Роздільність, розкладка меню і шрифти підставляються автоматично
(на 128×128 — більший шрифт і більше рядків).

**КРОК 2a — I2C-дисплеї.** `SDA=21`, `SCL=22`, адреса `0x3C`. Працюють на
будь-яких GPIO.
- `#define DISPLAY_HW_I2C` — апаратний I2C (рендер у ~10× швидше).
  **Якщо апаратна шина не відповідає, прошивка САМА переходить на програмний
  I2C на тих самих пінах** — дисплей працює завжди; у Serial (115200) буде
  `HW I2C OK` або `SW I2C fallback`. Часта причина відмови HW I2C — слабкі
  підтяжки: поставте зовнішні **4.7 кОм** з SDA і SCL на 3.3V.
- `#define DISPLAY_I2C_KHZ 0` — частота шини, `0` = АВТО (безпечний максимум
  з драйвера: SSD1306/SH1106/SH1107 — 400 кГц, SSD1327 — лише 100 кГц).

**КРОК 2b — SPI-дисплеї** (ST7567 / Nokia 5110). Апаратний SPI ESP32:
`SCK=18`, `MOSI=23` (до CLK/DIN), керуючі `CS=5`, `DC=17`, `RST=16`.
- `DISPLAY_ST7567_XOFF` (за замовч. 4) — зсув картинки, підберіть 0…4, якщо
  зсунута.
- `#define DISPLAY_CONTRAST 128` (опційно) — ручний контраст (для ST7567
  більше = темніше, типово 30…90).

> GME128128-02 буває на SH1107 **або** SSD1327 — якщо картинка спотворена,
> перемкніть між цими двома опціями. На Nokia 5110 (84×48) частина нижніх
> рядків не влазить — усі дані краще дивитись у вебі.

## 4. Прошивка

1. Оберіть плату ESP32 і порт (*Tools → Board / Port*).
2. Скомпілюйте та завантажте скетч `motorola-battery-reader-web.ino`.
3. **Завантажте файли у SPIFFS** — вміст теки `data/` (`index.html`,
   `logo.png`) через *Tools → ESP32 Sketch Data Upload* (плагін ESP32FS для
   Arduino IDE 1.8.x). **Без цього кроку веб-сторінка не відкриється.**
   - Помилка `No port specified` = не вибрано порт у *Tools → Port* або
     відкритий Serial Monitor (закрийте його).

---

## 5. Перше під'єднання

1. Підключіться до Wi-Fi **`BatteryReader_moto`**, пароль **`12345678`**.
2. **Captive-portal:** телефон/ПК сам запропонує «Вхід у мережу» і відкриє
   сторінку. Якщо ні — відкрийте `http://192.168.4.1`.
3. Пароль для будь-якого запису/скидання — **`admin123`**
   (`ADMIN_PASSWORD` у `settings.h`).

## 6. Веб-інтерфейс

### Вкладка «Огляд»
Заряд (%, з джерелом ICA/напруга), напруга, температура, модель, ємність,
знос, кількість циклів заряд/розряд, справжність (OK / РИЗИК + причина).
Кнопки: **Зчитати АКБ**, **Оновити**.

### Вкладка «Дані»
- Модель і серійний номер чипа (лазерний ROM DS2438).
- DS2438: напруга, струм (мА), температура, **Залишок (ICA)**,
  **Заряджено (CCA)**, **Розряджено (DCA)** — усе в мА·год і циклах,
  з сирим значенням у дужках.
- Повні hex-дампи DS2438 (64 Б) і DS2433 (512 Б).

### Вкладка «Прошивка»
1. **Зчитування та резервна копія** — зчитати АКБ, завантажити дампи
   DS2433/DS2438 у `.bin`. **Завжди робіть бекап перед записом.**
2. **Запис / відновлення прошивки** — записати `.bin` (512 Б) у DS2433.
   Це і є відновлення повністю стертої АКБ: запис **завідомо справного**
   дампа тієї ж моделі.
3. **Запис DS2438** — записати `.bin` (64 Б) у монітор.
4. **♻️ Скидання для рекалібрування** — обнуляє лічильники заряду/розряду
   (CCA/DCA/таймер) і знос (ємність→100 %), перераховує суми, пише в чіп.
   Далі калібруйте АКБ на **оригінальному** IMPRES-ЗП.
5. **🔋 Змінити ємність (знос)** — ввести 0–100 %, запис у запис історії 0x17
   + виправлення суми → у чіп.
6. **🛠 Ремонт цілісності** — перерахунок контрольної суми заголовка
   (Σ≡0x41), сум відомих записів (Σ≡0x5A) і відновлення дзеркала калібрування
   з уцілілого DS2438 (`DS2438[24:50]→DS2433[1:27]`). Статус показано:
   заголовок ✅/❌, дзеркало ✅/❌.
7. **✏️ Редактор байтів (експертний)** — правка будь-яких байт DS2433/DS2438
   прямо в браузері. «Автовиправлення» перераховує суму заголовка DS2433 і
   тримає дзеркало узгодженим. Кнопка «Завантажити» підтягує поточний дамп.

> Усі незворотні операції — під паролем. Спочатку резервна копія.

## 7. Керування на пристрої (кнопки)

Сторінки меню (перелистування кнопкою «Вперед»):
1. Головна (заряд + статус) 2. Модель / серійний 3. Дані батареї (DS2438)
4. Стан АКБ (ємність/знос/цикли/справжність) 5. Дамп DS2438 (hex)
6. Дамп DS2433 (hex) 7. Скидання.

- **«Вперед» коротко** — наступна сторінка.
- **«Вперед» довго (0.8 с)** — повторне зчитування АКБ.
- **«Назад» коротко** — попередня сторінка; на сторінці «Скидання» —
  взвід, друге натискання — підтвердження скидання.

## 8. Індикація світлодіодами (`leds.h`)

| Стан | Сигнал |
|---|---|
| Очікування | короткий зелений пульс раз на 3 с |
| Читання чипа | зелений мигає ~3 Гц |
| Запис у чип | зелений/червоний **поперемінно** (не відключати!) |
| Успіх | зелений горить ~1.2 с |
| Помилка | 4 швидких червоних мигання |
| Фатальний збій старту | постійний червоний |

Індикація неблокуюча — не гальмує кнопки і веб.

## 9. Одиниці вимірювання

- **Лічильники заряду — у мА·год:** `мА·год = регістр × 0.4882 / Rsense`.
  Резистор струму: `settings.h → DS2438_RSENSE_OHM` (за замовч. 0.025 Ом) —
  підберіть, щоб ємність сходилась з паспортною (типово 20…40 мОм).
- **Цикли:** 1 цикл = сумарний заряд (розряд), рівний одній повній
  паспортній ємності. Ємність АКБ: `settings.h → BATTERY_RATED_MAH`
  (за замовч. 2500; PMNN4409 — 2500, PMNN4493 — 3000).
- **Заряд, %:** пріоритет ICA (якщо DS2438 рахує струм, IAD=1), інакше
  оцінка за напругою (`BATTERY_EMPTY_MV`…`BATTERY_FULL_MV`).

## 10. Формат прошивки IMPRES (за аналізом дампів)

- **Контрольна сума заголовка DS2433:** `сума 0x00..0x1F ≡ 0x41`
  (байт 0x1F — контрольний).
- **Контрольна сума TLV-записів:** сума байт запису разом з її контрольним
  байтом `≡ 0x5A` (напр. запис історії ємності 0x17).
- **Дзеркало калібрування:** `DS2438[24:50] == DS2433[1:27]` на кожній
  справжній АКБ. Блок **однаковий для всіх батарей однієї моделі**
  (4488A і 4493A збігаються, 4409A відрізняється).
- **Модель** — в авторитетному записі `0x0B` (`0B 'P' ...`), а також
  дублюється як ASCII у заголовку.

**Прив'язки до серійного номера чипа НЕМАЄ.** Побайтове порівняння двох
батарей однієї моделі показує: усі відмінні байти — лічильники
використання/телеметрія; підпису від лазерного ROM DS2438 у даних немає.
Прошивка прив'язана до **МОДЕЛІ**, тому відновлення можливе з еталона тієї
ж моделі без підбору серійника.

## 11. Типові сценарії

**Відновити повністю стерту АКБ (DS2433 = нулі, DS2438 цілий):**
запишіть `.bin` еталонної батареї тієї ж моделі (вкладка «Прошивка» →
«Запис/відновлення»), потім «🛠 Ремонт цілісності» — він синхронізує дзеркало
з уцілілого DS2438 цієї батареї.

**Пошкоджений заголовок/калібрування (рація «не бачить»):**
«🛠 Ремонт цілісності» — перерахує суми і відновить дзеркало.

**Оригінал «невідомий» або не вмикається:** якщо суми/дзеркало в нормі —
проблема не в прошивці, а в елементах (розряджені/мертві) або потрібне
калібрування на IMPRES-ЗП. Спробуйте «♻️ Скидання для рекалібрування».

**Клон з обнуленим DS2438** (напр. PT4409A): рація відхиляє через відсутність
калібрування; повне відновлення потребує еталонного DS2438 тієї ж моделі.

**Змінити показник ємності:** «🔋 Змінити ємність» → введіть % → запис.

**Точкова правка будь-якого байта:** «✏️ Редактор байтів» → «Завантажити» →
правка → «Записати байти» (з увімкненим автовиправленням сум).

> ⚠️ **Автоматична «розлочка» без еталона неможлива** — калібрувальні дані
> унікальні для моделі й не виводяться математично. Перед будь-яким записом
> робіть резервну копію.

## 13. Робота по USB (Windows-клієнт)

Пристрій дублює весь функціонал вебу через **USB-кабель** (той самий, яким
прошиваєте). Працює **паралельно** з Wi-Fi.

**Клієнт — `client_usb.html`** (Web Serial):
1. Відкрийте `client_usb.html` у **Chrome** або **Edge**.
   - Якщо сторінка пише, що Web Serial недоступний з `file://`, покладіть файл
     у теку і запустіть `python -m http.server`, відкрийте
     `http://localhost:8000/client_usb.html`.
2. Натисніть **«Підключити»** → оберіть COM-порт ESP32 (той самий, що в
   Arduino IDE). Зачекайте ~1.5 с (ESP32 перезавантажується при відкритті
   порту — це нормально).
3. Далі — той самий інтерфейс, що у вебі: вкладки Огляд / Дані / Прошивка,
   зчитування, дампи, запис, скидання, ремонт, зміна ємності, редактор байтів.

**Протокол** (`serial_api.h`, 115200 бод) — текстові команди, відповідь одним
рядком `#R#{json}`; підходить і для власного клієнта (Python/C#):

| Команда | Дія |
|---|---|
| `PING` | ідентифікація |
| `READ` | зчитати обидві мікросхеми |
| `INFO` | усі декодовані поля + дампи (hex) |
| `GET33` / `GET38` | сирий дамп (hex) |
| `WRITE33 <hex>` / `WRITE38 <hex>` | запис дампа (як є) |
| `WRITEFIX33 <hex>` | запис DS2433 + автопочинка суми/дзеркала |
| `RESET` / `REPAIR` | скидання / ремонт цілісності |
| `SETCAP <0..100>` | змінити ємність % |

> Пароль по USB не потрібен (фізичний доступ = авторизація). Відладкові рядки
> у Serial не мають префікса `#R#` — клієнт їх ігнорує.
